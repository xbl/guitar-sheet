# 练习模式：节拍器 + 谱面自动滚动 — 设计规格

**状态**：草案（与产品负责人对齐用）  
**日期**：2026-05-02  
**依赖**：[Guitar Sheet MVP 总规](./2026-05-02-guitar-sheet-design.md) 中的「阅读视图」；本规格描述在阅读视图或等价谱面容器上叠加的练习辅助能力。  
**范围**：第一期练习辅助（不引入 Guitar Pro / MusicXML；不与 BPM 做复杂联动）。

---

## 1. 背景与目标

### 1.1 用户价值

初学者**双手弹琴**时难以手动拖谱。需要在「跟谱练」时：**一键播放**后谱面**自动向下滚动**；可选**节拍器**打拍；滚动快慢可单独调节；节拍器可**静音**而不影响滚动。

### 1.2 成功标准

- 用户在谱面区域内点击 **播放** 后，谱面在可滚动容器内**持续向下滚动**直至底部或用户暂停。  
- 用户可通过 **1～20 档**预置速度调节垂直滚动快慢（滑杆选档）。  
- 节拍器按用户设定的 **BPM** 工作；提供 **静音**，静音时**滚动行为与有声时一致**（仅关闭滴答声输出）。  
- 滚动速度与 BPM **不强制数学联动**（简单版：档位只映射垂直速度；BPM 只驱动节拍器）。

---

## 2. 显式非目标（本期）

- 滚动与乐谱小节线、和弦变换的**语义对齐**（需谱面结构信息，本期不做）。  
- 音频伴奏同步、MIDI 回放。  
- 节拍器复杂拍号（本期可采用**固定四分音符拍**感知；若实现极简，仅表现为稳定 BPM 脉冲即可）。

---

## 3. 用户可见行为

### 3.1 控件

| 控件 | 行为 |
|------|------|
| **播放 / 暂停** | 切换「进行中」状态。进行中：开启自动滚动；节拍器按 BPM 发声（若未静音）。暂停：滚动与节拍器发声均停止；**不从顶部复位**（无单独「停止」按钮）。 |
| **节拍器 BPM** | 调节节拍器间隔；范围建议在实现中固定为合理区间（如 40–240），具体数字在实现计划里写死。 |
| **节拍器静音** | 开关；**仅**关闭节拍器音频输出；**不**暂停滚动、不改变档位与 BPM 设定。 |
| **滚动速度** | **横向滑杆**，取值 **整数 1～20**（离散档）；拖动立即生效；播放中改档则**立即**按新速率滚动。 |

### 3.2 默认值（建议）

- 滚动档位：**10**（中间档）。  
- BPM：**120**。  
- 节拍器静音：**关**（有声）。

### 3.3 底部行为

- 当滚动容器到达**底部**（无可再向下滚动）时：**自动退出「播放」状态**（与手动暂停一致：滚动停、节拍器停），避免在底部空转。

### 3.4 「回到顶部」

- **MVP 不强制**专用按钮；用户可手动拖动滚动条回到顶部。若实现阶段交互评审需要，可增加次要「回到顶部」动作（实现计划可选）。

## 4. 技术设计

### 4.1 架构（前端为主）

- **谱面容器**：`overflow-y: auto`（或等价）的可滚动元素；由 `ref` 持有 DOM。  
- **`useAutoScroll`**（Composable）：输入 `isPlaying`、`speedLevel`（1–20）、`scrollParentRef`；使用 `requestAnimationFrame`（或等效循环）按映射表更新 `scrollTop`；每帧检测是否触底并触发「停止播放」回调。  
- **`useMetronome`**（Composable）：输入 `bpm`、`muted`、`isPlaying`；使用 **Web Audio API**（`AudioContext`）调度节拍脉冲；`muted === true` 时将输出增益置 **0**（或不调用发声分支），**不改变** `isPlaying` 与滚动逻辑。  
- **映射函数**：`speedLevel → pixelsPerSecond`（或 `deltaY` 上限），**单调递增**；1 最慢、20 最快；曲线（线性或分段线性）在实现计划中给出常数表与单一数据源，便于单测。

### 4.2 BPM 与滚动

- **解耦**：滚动速率**仅**由 1～20 档决定；BPM **仅**影响节拍器。本期不要求两者公式关联。

### 4.3 持久化（建议）

- 使用 `localStorage`（或应用后续统一设置存储）持久化：**BPM**、**滚动档位**、**节拍器静音**，下次启动恢复。

### 4.4 错误与边界

- **音频受限环境**：若 `AudioContext` 处于 `suspended`，在用户首次点击播放时尝试 `resume()`；仍失败则节拍器静音并在 UI 提示（不阻断滚动）。  
- **窗口隐藏**：可选策略为暂停播放（节省资源）；若第一期不实现，须在实现计划中注明「未处理后台暂停」。  
- **容器尺寸变化**：滚动位置自然保留；触底判定基于当前 `scrollHeight` / `clientHeight`。

---

## 5. 测试策略

- **单元测试**：`speedLevel → px/s`（或每帧 delta）映射、触底判定纯函数（给定 scrollTop、scrollHeight、clientHeight、delta）。  
- **手动**：播放、暂停、静音、改档、触底停播、持久化恢复。

## 7. 自检（成文时）

- **占位符**：无 `TBD`。后台策略若未定，已标明「可选 / 实现计划」。  
- **一致性**：静音不关滚动；触底停播；仅播放/暂停无第三停止态。

- **歧义**：「静音」仅指节拍器音频；档位为整数 1～20。
